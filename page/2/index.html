异常与析构

发表于 2016-02-27 更新于 2021-08-16 分类于 tech

在C++中，禁止在析构函数中抛出异常，被认为是一条铁律，何以见得呢？
自C++11开始，析构函数默认声明为exception free，也就是不允许抛出任何异常。同时STL的所有容器也遵循此律，如果你传给容器的对象在析构中抛出异常，程序立即会被terminate，因为容器的析构函数不会吞掉这个异常，它会向外传出，而这违背了容器析构函数的exception free的声明，程序立即被终止运行。

那么这条铁律的背后原委究竟是什么呢。

发表于 2016-01-25 更新于 2021-08-16 分类于 tech

debug一般有两种形式，使用调试工具断点单步, 或者分析trace信息。
这里说的trace包括调用栈，调用参数等运行时的上下文信息，它们可以在不影响功能地前提下从实际环境中无痛收集。这两种形式就好比一个是开膛破肚的外科手术，一个是望闻问切的推理分析。前者固然简单直观，但一般仅限于开发环境，如果问题只出现于实际环境，那就只能借助trace来找出真凶。
因此，错误处理的要求之一就是提供尽可能多的上下文信息，以便快速定位问题。
